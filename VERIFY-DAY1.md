# VERIFY-DAY1 — 1일차 전 채널 발행 직접 검증 (새 세션용)

> 이 세션의 임무: **코드만 보지 말고 직접 실행**해서 "1일차에 전 채널이 진짜 다 되는지" 검증하고, 빠진 건 채운다.
> 먼저 `BUILD-SPEC.md`(§0 ⚠️정정 포함) 읽기. 그다음 이 문서.
>
> ⚠️ **환경:** 플러그인 구독 디스패치(`melanoir-copywriter` 에이전트)를 실제로 검증하려면
> **Claude Desktop에 이 플러그인이 설치된 상태**여야 한다(`/plugin` 로 melanoir-studio 로드). 그래야 `/melanoir-daily`가 구독 LLM으로 진짜 골드 카피를 뽑는지 확인 가능.

---

## A. 이미 확인된 것 (옛 세션이 직접 점검함 — 재확인만)

| 항목 | 결과 |
|------|------|
| 배포 콘솔 `web-...vercel.app` | ✅ 동작 (Supabase 연결·골드 카피·검수 0/0·콘솔 에러 0). "연결 확인 중…"은 JS 실행 전 초기상태 아티팩트 |
| `npm run verify` 전체 | ✅ green (render 1.64%·guard 22/0·image 10/0·self-check 24/0·e2e 9/0) |

→ 엔진·렌더·가드·인프라는 탄탄. **문제는 채널 커버리지.**

## B. 🚩 확인된 갭 (직접 고칠 것)

**`daily.mjs` + `melanoir-daily.md`는 토픽 1개 → [IG 캐러셀] + [자사몰 인사이트카드] 2개만 만든다.**
**LinkedIn · Threads · 블로그(naver-blog) 채널 카피 생성이 통째로 없다.** (grep: 채널 토큰 0건, insight-card만 존재)

원인: BUILD-SPEC이 IG 캐러셀 중심이라 멀티채널 누락. 옛 `../마케팅 자동화 에이전트/`는 채널별 copywriter로 IG/LinkedIn/Threads/naver-blog 카피를 다 만들었다 — 그 패턴을 참고해 **채널 최적화 카피 생성**을 추가한다.

---

## C. 검증 체크리스트 (전부 직접 실행 + 증거 남기기)

### ① 채널 1일차 발행 — 목표 5/5 (현재 2/5)
`/melanoir-daily "토픽"` 한 번에 아래가 **전부** 골드 품질로 나오는지:
- [ ] **IG 캐러셀** (7~10장 PNG + 캡션) — 현재 ✅
- [ ] **자사몰 인사이트카드** (`web/site/insights/` 스테이징) — 현재 ✅
- [ ] **LinkedIn** 카피 — ❌ 추가 필요 (B2B 톤, 캐러셀 thesis 기반 장문)
- [ ] **Threads** 카피 — ❌ 추가 필요 (짧은 후킹)
- [ ] **블로그(naver-blog)** 본문 — ❌ 추가 필요 (긴 글 + 섹션)
> 각 채널은 **같은 토픽 thesis**에서 파생하되 채널 포맷/톤에 맞게. 브랜드락·광고법은 5채널 모두 `guard` 통과 필수.

### ② 하네스 엔지니어 세팅
- [ ] `npm run verify` green 재확인
- [ ] 멀티채널 추가 후에도 verify·guard 깨지지 않는지

### ③ 피드백 루프 + 플러그인 연동 (구독)
- [ ] 콘솔에서 피드백 저장 → `npm run pull` → `npm run distill` → `learnings/`에 규칙 누적되는지 (실제 라운드트립, Supabase 키 필요)
- [ ] 그 learnings가 **다음 생성 brief에 주입**되는지 (생성 입력에 반영 확인)
- [ ] **`/melanoir-daily`가 Claude Desktop 구독에서 `melanoir-copywriter` 에이전트를 실제로 디스패치**해 골드 카피를 쓰는지 (스켈레톤 아님). ← 이게 "구독 플러그인 연동"의 핵심

---

## D. 완료 기준
1. `/melanoir-daily "아무 토픽"` → **5채널 + 자사몰 인사이트** 전부 골드 품질 산출 (guard 통과).
2. 피드백 → distill → 다음 생성 주입까지 **실제 실행**으로 한 바퀴 증명.
3. 플러그인 디스패치가 **구독으로**(API 키 없이) 골드 카피를 뽑음을 확인.
4. 각 단계 증거(샘플 출력·로그) 남기고, 새로 발견한 비자명 사항은 메모리에 기록.

## E. 절대 규칙
- 직접 실행 검증(사장님 룰). 렌더는 골드 대조, 전송/실시간/디스패치는 실제 라운드트립.
- 멀티채널 추가해도 **골드 품질·브랜드락·레이어 분리**는 IG와 동일하게 강제.
- 생성 LLM = **구독(copywriter 에이전트), Anthropic API 직접 호출 금지** (BUILD-SPEC §0 정정).
